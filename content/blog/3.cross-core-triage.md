# Cross-Core Triage: Running AzerothCore, TrinityCore, and MaNGOS Side-by-Side

- Identify whether an issue is core-specific or shared.
- Validate fixes by comparing before/after across cores.
- Reduce duplicated effort by reusing proven solutions.
- Improve triage speed with consistent environments.

## Port and Realm Isolation Strategy

When running multiple worldservers, ensure ports and realm entries do not conflict.

Example ports:
| Service | Default | AC 1 | AC 2 | TC | CM |
| ---------------------- | ------------ | ------ | ------ | ------ | ------ |
| DB | 3306 | 3311 | 3312 | 3315 | 3316 |
| World | 8085 | 8085 | 8086 | 8087 | 8088 |
| Auth | 3724 | 3724 | 3725 | 3726 | 3727 |
| realmlist (DB) | 8085 | 8085 | 8086 | 8087 | 8088 |
| Client realmlist.wtf | 3724 | 3724 | 3725 | 3726 | 3727 |

These values live in each core's `worldserver.conf`, `authserver.conf`, and the authentication DB (e.g., `acore_auth.realmlist`). Match your client `realmlist.wtf` to the auth port in use (e.g., `set realmlist 127.0.0.1:3725`).

## Other Cores Setup

- Use containers, VMs, or Distrobox to avoid dependency conflicts.
- Keep each core isolated: separate DBs, ports, and build directories.
- Prefer read-only reference DBs for clean comparisons.

### TrinityCore 3.3.5 (Distrobox)

[TrinityCore - Linux Core Installation](https://trinitycore.info/install/Core-Installation/linux-core-installation)

Use a containerized DB and initialize with:

```sql
CREATE USER 'trinity'@'%' IDENTIFIED BY 'trinity';

CREATE DATABASE world;
CREATE DATABASE characters;
CREATE DATABASE auth;

GRANT ALL PRIVILEGES ON world.* TO 'trinity'@'%';
GRANT ALL PRIVILEGES ON characters.* TO 'trinity'@'%';
GRANT ALL PRIVILEGES ON auth.* TO 'trinity'@'%';
```

Build inside a [Distrobox](https://distrobox.it) for isolation. See `distrobox.ini` in your notes/gists.

### CMaNGOS (Ubuntu 22.04 VM)

[CMaNGOS Installation Guide](https://github.com/cmangos/issues/wiki/Installation-Instructions)

1. Create an Ubuntu 22.04 VM.
2. Allow external MySQL connections (bind-address `0.0.0.0`).
3. Restart MySQL and connect from the host.
